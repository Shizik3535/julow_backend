# TaskFlow — Система управления задачами и проектами

## Описание продукта

TaskFlow — платформа для управления задачами и проектами, предназначенная для широкого спектра пользователей: от индивидуальных разработчиков до военных предприятий и государственного сектора.

Система разворачивается как:
- **SaaS** — облачное решение для индивидуальных пользователей и команд
- **Self-hosted** — Docker/Kubernetes для бизнеса
- **On-premise** — bare-metal/закрытый контур для гос. сектора и военных предприятий

## Целевая аудитория

| Сегмент | Описание | Тариф | Deployment |
|---------|----------|-------|------------|
| **Personal** | Индивидуальные пользователи, фрилансеры | Free | SaaS |
| **Teams** | Команды до 25-50 человек | Start / Basics | SaaS |
| **Business** | Компании до 100-500 человек | Professional / Business | SaaS / Self-hosted |
| **Enterprise** | Корпорации, гос. структуры, военные предприятия | Enterprise | Self-hosted / On-premise |

### Зависимости между контекстами

| Контекст | Зависит от | Тип связи |
|----------|-----------|-----------|
| Organization | Identity | Customer-Supplier |
| Workspace | Identity, Organization | Customer-Supplier |
| Project | Workspace | Customer-Supplier |
| Task | Project, Workspace | Customer-Supplier |
| Sprint | Project | Customer-Supplier |
| Comment | Task, Identity | Customer-Supplier |
| File | Task, Comment, Workspace | Customer-Supplier |
| Time Tracking | Task, Identity | Customer-Supplier |
| Notification | все контексты | Subscriber |
| Reporting | Task, Project, Time Tracking | Subscriber |
| Billing | Identity, Organization, Workspace | Customer-Supplier |
| Search | Task, Project, Comment, File | Subscriber |
| Administration | все контексты | Conformist |

## Глоссарий

| Термин | Описание |
|--------|----------|
| **Aggregate** | Кластер доменных объектов, управляемых как единое целое через корневую сущность |
| **Bounded Context** | Изолированный домен со своим Ubiquitous Language |
| **Entity** | Объект с уникальной идентичностью (ID) |
| **Value Object** | Объект без идентичности, определяемый значениями |
| **Domain Event** | Факт, произошедший в домене, который может быть интересен другим контекстам |
| **Organization** | Юридическое лицо / компания — верхний уровень мульти-тенантности |
| **Workspace** | Рабочее пространство внутри организации или standalone (для Personal/Teams) |
| **Project** | Проект внутри workspace с задачами, методологией и участниками |
| **Sprint** | Временной отрезок в Scrum-методологии для выполнения набора задач |
| **Epic** | Крупная пользовательская история, объединяющая несколько Story |
| **Story** | Пользовательская история, описывающая функционал с точки зрения пользователя |
| **Task** | Единица работы внутри проекта |
| **Subtask** | Подзадача, декомпозиция Task |
| **Workflow** | Набор статусов и переходов между ними (state machine) |
| **Swimlane** | Горизонтальное разделение на Kanban-доске (по исполнителю, приоритету и т.д.) |
| **WIP Limit** | Ограничение количества задач в колонке (Work In Progress) |
| **Tenant** | Изолированная единица данных (Organization или standalone Workspace) |
| **SSO** | Single Sign-On — единая точка входа через корпоративный Identity Provider |
| **SAML 2.0** | Протокол обмена аутентификационными данными между IdP и SP |
| **TOTP** | Time-based One-Time Password — временной одноразовый пароль для 2FA |
| **Audit Log** | Журнал аудита — запись всех действий пользователей в системе |
| **Data Residency** | Правило хранения данных в определённом географическом регионе |
| **Grace Period** | Период после неоплаты, в течение которого данные сохраняются |
